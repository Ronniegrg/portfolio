## ğŸ“¦ Installation

1. **Clone the repository:**

   ```bash
   git clone https://github.com/Ronniegrg/portfolio.git
   cd portfolio
   ```

2. **Install dependencies:**

   ```bash
   npm install
   # or for environments with symlink issues:
   npm run install-safe
   ```

3. **Set up environment variables:**

   - Create a `.env` file in the project root:

   ```env
   VITE_EMAILJS_PUBLIC_KEY=your_emailjs_public_key
   ```

   - Get your EmailJS public key from [EmailJS Dashboard](https://www.emailjs.com/)
   - Create an email template with variables: `{{name}}`, `{{email}}`, `{{subject}}`, `{{message}}`

4. **Development server:**

   ```bash
   npm run dev
   ```

5. **Build for production:**

   ```bash
   npm run build
   ```

6. **Preview production build:**
   ```bash
   npm run preview
   ```

### Available Scripts

- `npm run dev` - Start development server with hot reload
- `npm run build` - Build for production with optimizations
- `npm run preview` - Preview production build locally
- `npm run lint` - Run ESLint for code quality checks
- `npm run clean` - Clear Vite cache
- `npm run setup` - Full setup with cache clearing
- `npm run deploy` - Deploy to GitHub Pages
- `npm run deploy-force` - Force deploy without history

## ğŸ”’ Environment Variables

### Required Configuration

```env
# EmailJS Configuration (Required for contact form)
VITE_EMAILJS_PUBLIC_KEY=your_emailjs_public_key_here

# Optional: Analytics Configuration
VITE_ANALYTICS_ID=your_analytics_site_id
```

### Security Best Practices

- **Never commit `.env` files** - Already included in `.gitignore`
- **Use VITE\_ prefix** - Only variables with this prefix are exposed to the client
- **Rotate keys regularly** - Update EmailJS keys periodically for security
- **Environment-specific configs** - Use different keys for development and production

### EmailJS Setup

1. Create account at [EmailJS.com](https://www.emailjs.com/)
2. Set up email service (Gmail, Outlook, etc.)
3. Create email template with variables: `{{name}}`, `{{email}}`, `{{subject}}`, `{{message}}`
4. Copy public key from dashboard to `.env` file
